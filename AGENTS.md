# ModelForge Agent Workflow Addendum

## Commit Discipline

- Commit after meaningful implementation changes instead of letting work pile up.
- When adding Graphiti memory or updating progress/rules, commit any corresponding repo file changes in the same workflow.
- Before committing, run the relevant focused checks for the changed surface when practical.

## Review Workflow

- Use frequent valid remote PRs when a feature branch is ready for review.
- CodeRabbit PR review is part of the normal workflow.
- When CodeRabbit review feedback arrives, use the CodeRabbit review skill to inspect and fix relevant actionable issues before continuing.
